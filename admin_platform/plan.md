# 관리자 플랫폼 개선 계획

## 1. 파일 첨부 기능 추가

- [ ] **모델 수정**: 데이터베이스에 파일 경로를 저장할 수 있도록 `Request` 모델에 `file_path` 컬럼을 추가합니다.
- [ ] **폼 수정**: 파일 입력을 받을 수 있도록 요청 생성/수정 템플릿(`business_trip_form.html`, `overtime_form.html` 등)에 `<input type="file">` 필드를 추가합니다.
- [ ] **라우트 수정**: 파일 업로드를 처리하고 저장하는 로직을 `request_routes.py`에 추가합니다. 업로드된 파일은 `documents/` 디렉토리에 저장합니다.
- [ ] **상세 보기 수정**: 업로드된 파일을 다운로드하거나 볼 수 있도록 요청 상세 보기 페이지에 링크를 추가합니다.

## 2. 모바일 최적화

- [ ] **CSS 수정**: 미디어 쿼리를 사용하여 모바일 화면 크기에 맞게 레이아웃이 조정되도록 `style.css`를 수정합니다.
- [ ] **템플릿 검토**: 모든 템플릿 파일이 모바일에서 정상적으로 보이는지 확인하고 필요한 경우 수정합니다.

## 3. UI 개선

- [ ] **디자인 시스템 도입**: 일관된 디자인을 위해 간단한 디자인 시스템(색상, 폰트, 컴포넌트 스타일 등)을 정의하고 `style.css`에 반영합니다.
- [ ] **템플릿 리팩토링**: 정의된 디자인 시스템에 맞게 모든 템플릿의 HTML 구조와 클래스를 수정합니다.

## 4. 검색 기능 개선

- [ ] **검색 폼 추가**: `approve_list.html`, `my_requests.html` 등 목록 페이지에 기간, 요청 유형, 직원 이름 등으로 검색할 수 있는 폼을 추가합니다.
- [ ] **라우트 수정**: 검색 조건에 따라 데이터를 필터링하는 로직을 관련 라우트 함수에 추가합니다.
